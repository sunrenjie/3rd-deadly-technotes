   5     Improve the API design of Http2OrHttpChooser and SpdyOrHttpChooser
   6 
   7     Related: #3641 and #3813
   8 
   9     Motivation:
  10 
  11     When setting up an HTTP/1 or HTTP/2 (or SPDY) pipeline, a user usually
  12     ends up with adding arbitrary set of handlers.
  13 
  14     Http2OrHttpChooser and SpdyOrHttpChooser have two abstract methods
  15     (create*Handler()) that expect a user to return a single handler, and
  16     also have add*Handlers() methods that add the handler returned by
  17     create*Handler() to the pipeline as well as the pre-defined set of
  18     handlers.
  19 
  20     The problem is, some users (read: I) don't need all of them or the
  21     user wants to add more than one handler. For example, take a look at
  22     io.netty.example.http2.tiles.Http2OrHttpHandler, which works around
  23     this issue by overriding addHttp2Handlers() and making
  24     createHttp2RequestHandler() a no-op.
  
We are urged to analyze the examples more thoroughly to understand the whole thing. :)

  25 
  26     Modifications:
  27 
  28     - Replace add*Handlers() and create*Handler() with configure*()
  29     - Rename getProtocol() to selectProtocol() to make what it does clear
  30     - Provide the default implementation of selectProtocol()
  31     - Remove SelectedProtocol.UNKNOWN and use null instead, because
  32       'UNKNOWN' is not a protocol
  33     - Proper exception handling in the *OrHttpChooser so that the
  34       exception is logged and the connection is closed when failed to
  35       select a protocol
  36     - Make SpdyClient example always use SSL. It was always using SSL
  37       anyway.
  38     - Implement SslHandshakeCompletionEvent.toString() for debuggability
  39     - Remove an orphaned class: JettyNpnSslSession
  40     - Add SslHandler.applicationProtocol() to get the name of the
  41       application protocol
  42       - SSLSession.getProtocol() now returns transport-layer protocol name
  43         only, so that it conforms to its contract.
  
Plenty of bonus ;)

  44 
  45     Result:
  46 
  47     - *OrHttpChooser have better API.
  48     - *OrHttpChooser handle protocol selection failure properly.
  49     - SSLSession.getProtocol() now conforms to its contract.
  50     - SpdyClient example works with SpdyServer example out of the box
